À propos de Santé Data France

Santé Data France est une application web développée dans le cadre de la SAE 2.01. Elle permet d’explorer, visualiser et analyser des données publiques de santé issues de l’Assurance Maladie.

Contexte du projet

Le projet s’inscrit dans la continuité de la SAE 2.04, où une base de données relationnelle a été construite à partir de dimensions issues des données de santé. La SAE 2.01 consiste ensuite à exploiter cette base et l’API Ameli dans une application web.

📌

Objectifs

  • Créer une interface web claire et simple à utiliser.
  • Permettre la recherche par profession, région, département et année.
  • Afficher les résultats sous forme de tableaux, graphiques et cartes.
  • Comparer des données de santé entre plusieurs territoires ou périodes.
  • Rendre l’analyse accessible à un utilisateur non technique.
🧰

Technologies utilisées

Python Flask Jinja2 SQLAlchemy MySQL HTML CSS JavaScript Chart.js Leaflet Git / GitHub
📊

Sources de données

Les données utilisées proviennent principalement de l’API publique data.ameli.fr, mise à disposition par l’Assurance Maladie.

Les données affichées concernent notamment les effectifs de professionnels de santé, les densités, les honoraires, les prescriptions et les indicateurs territoriaux.

Organisation technique

Architecture de l’application

Le projet suit une organisation proche du modèle MVC afin de séparer clairement les responsabilités du code.

01

Controllers

Les contrôleurs gèrent les routes Flask. Ils récupèrent les choix de l’utilisateur, appellent les services nécessaires et renvoient les templates HTML.

02

Models

Les modèles représentent les tables de la base de données avec SQLAlchemy : régions, départements, professions, types de prescriptions, etc.

03

Services

Les services regroupent la logique métier, les appels à l’API Ameli et la préparation des données utilisées dans les graphiques et les tableaux.

04

Templates & Static

Les templates Jinja2 affichent les pages, tandis que les fichiers CSS et JavaScript gèrent l’apparence, les graphiques, les filtres dynamiques et les interactions.

Finalité du site

L’objectif final est de proposer une plateforme lisible, moderne et interactive, capable de transformer des données publiques complexes en informations compréhensibles. Le site aide ainsi à mieux observer les différences territoriales, les évolutions médicales et les tendances liées au système de santé français.
